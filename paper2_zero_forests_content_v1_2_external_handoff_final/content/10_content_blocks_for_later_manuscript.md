# Content blocks for later manuscript use

These are reusable blocks, not a manuscript.

## Block A. One-sentence contribution

Zero forests turn vanishing tensor obligations into replayable, domain-certified certificate objects whose raw proof routes remain auditable while their normalized evidence is route-independent.

## Block B. Definition payload

A zero forest contains:

```text
claim_id
object_signature
domain and assumptions
normalized_obligation
route-sensitive proof traces
route-independent normalized hash
formal_fragment label
trust_boundary label
```

## Block C. Seed result payload

```text
review_pass = true

normalized_hashes_recomputed_ok = 4
unique_normalized_certificates = 4
route_to_claim_compression = 8:4
```

## Block D. Guardrail payload

```text
The seed checker is Level-B evidence:
  tensor skeleton + replayed algebraic zero leaves.
It is not a full tensor CAS, not a complete formal proof, and not a universal zero solver.
```

## Block E. Negative-control payload

Tamper tests show that structural integrity claims should be divided into three gates:

```text
seed replay gate
content route-chain gate
domain semantic gate
```

## Block F. Formalization payload

The first formal kernel should not target the whole tensor pipeline.  It should target finite algebraic replay for rational/Laurent leaves, then attach tensor steps as explicitly labeled oracles until a stronger tensor kernel exists.
